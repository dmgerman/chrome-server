Chrome Server — Options

Edit the JSON config used to build context menus and dispatch Emacs-initiated requests. Hardcoded baseline menus (Org-capture and Org-roam-capture from selection) are always present and cannot be removed.

Raise Emacs

For each action, whether Emacs should be brought to the foreground. Capture / eww / youtube raise at the moment the buffer opens; save-page and save-chatgpt raise after the file has been written. The checkbox overrides whatever raise the menu's bundled config sets.

ActionRequestRaise?Default

Menus

Each entry adds a context-menu item. Fields: id, title, trigger (page|selection|link|image), request (WS request name), source (e.g. page-url, page-html, selection-text), optional raise, optional documentUrlPatterns.

Handlers (Emacs → browser)

Declarative bindings for chrome.* calls Emacs is allowed to invoke. Fields: name (WS request name), api (dotted chrome.* path), and one of args (literal), args-from: "payload", or args-shape (named adapter — e.g. user-script, focus-tab, tab-id).
