#+title: browsel — contributing
#+author: Daniel M. German

Reference for extending =browsel=: adding new menu items,
adding new browser → Emacs handlers, adding new Emacs → browser
handlers, or modifying the build pipeline.  Read =README.org= first
for what =browsel= does and how it's set up.

* The protocol

JSON text frames over a single WebSocket on =127.0.0.1:9130=.  Two shapes:

#+begin_src js
// Request
{ "id": "<uuid>", "name": "ORG_CAPTURE", "payload": { ... } }

// Response
{ "requestId": "<uuid>", "payload": { ... } }
#+end_src

Distinguished by which key is present.  Request names are
=SCREAMING_SNAKE_CASE=.  Error envelope inside a response payload:

#+begin_src js
{ "status": "error", "message": "..." }
#+end_src

Successful responses set =status: "ok"= and may include =message= (a
human-readable line that bubbles up to =*Messages*= and the popup).
Timeouts: =browsel-request-timeout= (default 10s) on the Emacs
side, 5s on the offscreen-doc side.

*Client identification.*  Every browser issues a =CLIENT_HELLO=
request as its first frame, with payload
={ client: "chrome", version: "0.8" }= or the Firefox equivalent.
Emacs records the name on the websocket and uses it to route
Emacs-to-browser requests when more than one browser is connected.
Until =CLIENT_HELLO= arrives the connection is tracked as
=unknown-N=.  If two clients announce the same name a =-2=, =-3=, …
suffix is appended.  Inspect the current roster with
=browsel-connected-clients=.

*Version gate.*  The =version= field in =CLIENT_HELLO= must match
=browsel-version= exactly (string equality).  A missing field,
empty string, or any value that differs causes the handler to reply
with ={ status: "error", message: "version mismatch: emacs=X
extension=Y" }= and leave the client unregistered.  The extension's
=ws-client.js= reads that reply, transitions to a terminal
=INCOMPATIBLE= status, closes the socket, and suppresses the
reconnect loop until the user clicks =Reconnect= in the popup (which
is itself useful only after rebuilding both sides).  The popup
displays "Version mismatch — rebuild and reload" in this state and
=chrome.action.setBadgeText= surfaces a ={!}= badge on the toolbar.
The policy is strict (no semver-style flex) because the protocol is
still pre-1.0 and any bump may be breaking.

** Request flow

Browser → Emacs (right-click, popup, hotkey, content-script relay):

1. Service worker gathers the payload (=payload.js= =gatherPayload=
   dispatches on the menu's =payload= field).
2. Service worker hands to the offscreen doc, which sends the WS frame.
3. Emacs's =:on-message= handler accumulates frames until complete,
   parses JSON, looks up the handler in =browsel--handlers=,
   invokes it.
4. Handler's return value becomes the response payload.

Emacs → browser:

1. =browsel-request= / =-async= generates a UUID, registers a
   callback, sends the frame.
2. Offscreen doc receives, asks the service worker to dispatch
   (=handlers.js= =dispatchEmacsRequest=).
3. Service worker resolves the =chrome.*= API via the handler entry's
   =api= path or =args-shape= adapter, returns the value.
4. Offscreen sends response; Emacs fires the callback.

** Frame fragmentation

WebSocket libraries call message callbacks /per-frame/, not per
message.  Big payloads (a page's HTML often runs >128 KB) come as one
initial frame plus several continuation frames.  =browsel.el=
buffers per-client via =browsel--rx-buffers= and only parses on
=websocket-frame-completep=.  Do not remove that — see
=ai/gotchas.md=.

* Configuration schema: =extension/config.json=

Top-level:

| Key              | Purpose                                                                                                                                                                                                                                                                                |
|------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| =extension=        | Manifest fields common to every target (name, version, icons, =options_ui=).                                                                                                                                                                                                             |
| =extensionTargets= | Per-target overlays.  =chrome= supplies MV3 fields (=service_worker=, =action=, =scripting=/=offscreen=/=userScripts= permissions, =host_permissions=); =firefox= supplies MV2 fields (persistent =background.scripts=, =browser_action=, =browser_specific_settings.gecko=, =<all_urls>= in =permissions=). |
| =menus=            | Right-click menu entries (each may carry a =command= hotkey).                                                                                                                                                                                                                            |
| =handlers=         | =chrome.*= calls Emacs is allowed to invoke (Emacs → browser).                                                                                                                                                                                                                           |
| =contentScripts=   | Declarative content scripts (matches/js/run_at).                                                                                                                                                                                                                                       |

** Menu entry

#+begin_src js
{
  "id":       "save-page",                  // stable identifier
  "title":    "Emacs: save page",
  "trigger":  "page",                       // page | selection | link | image
                                            // (or an array for multiple)
  "request":  "SAVE_PAGE",                  // WS request name
  "payload":  "page-html",                  // how to build the payload
  "raise":    false,                        // ask Emacs to focus?
  "documentUrlPatterns": ["https://..."],   // filter by page URL
  "targetUrlPatterns":   ["https://..."],   // filter by link/image URL
  "command":  {                             // optional Chrome command
    "name":          "save-page",
    "description":   "Save current page",
    "suggested_key": { "default": "Ctrl+Shift+S", "mac": "MacCtrl+Shift+S" }
  }
}
#+end_src

** =payload= kinds

A string (built-in) or an object (parameterised).

| String form               | Builds payload from                                       |
|---------------------------+-----------------------------------------------------------|
| =page-url=                | tab url + title                                           |
| =page-url-with-selection= | tab url + title + current selection                       |
| =selection-text=          | url + title + selection                                   |
| =page-html=               | url + title + main/article/body =innerHTML=               |
| =link-url=                | =info.linkUrl= + link text                                |
| =image-url=               | =info.srcUrl= + tab title                                 |
| =url=                     | context-aware: link / image / page URL, whichever applies |

Object kinds:

#+begin_src js
{ "kind": "tab-message", "method": "extract-conversation" }
#+end_src

The service worker calls =chrome.tabs.sendMessage(tabId, {method,
...})= and the matching content script replies with
=sendResponse({payload: ...})=.  Adding a new scraper is a content
script + this menu entry — no =payload.js= edit.

** Handler entry (Emacs → browser)

#+begin_src js
{ "name": "GET_ALL_TABS",  "api": "chrome.tabs.query",  "args": {} }
{ "name": "OPEN_TAB",      "api": "chrome.tabs.create", "args-from": "payload" }
{ "name": "FOCUS_TAB",     "api": "chrome.tabs.update", "args-shape": "focus-tab" }
{ "name": "EVAL_IN_ACTIVE_TAB",
  "api":  "chrome.userScripts.execute", "args-shape": "user-script" }
#+end_src

| Field         | Meaning                                                             |
|---------------+---------------------------------------------------------------------|
| =args=        | Static literal passed to the =chrome.*= API                         |
| =args-from=   | ="payload"= → request payload is passed as the API arg              |
| =args-shape=  | Named adapter in =extension/src/handlers.js=                        |

Built-in shape adapters: =focus-tab=, =tab-id=, =user-script= (the
last one is gated by =consent.js=).

** Content scripts

#+begin_src js
"contentScripts": [
  { "matches": ["<all_urls>"],            "js": ["src/content.js"],
    "run_at": "document_idle", "all_frames": false },
  { "matches": ["https://chatgpt.com/*"], "js": ["src/content-chatgpt.js"],
    "run_at": "document_idle" }
]
#+end_src

Adding a new scraper site: drop the JS in =extension/src/=, add a
=contentScripts= entry pointing at it, add a menu entry whose
=payload= is =tab-message= naming a method the script answers to.
Then add the file to =SRC_FILES= in the Makefile (=make lint= catches
imports of un-listed files but not contentScripts entries themselves).

* Built-in request catalogs

** Browser → Emacs

| Name               | Payload keys            | Effect                                            |
|--------------------+-------------------------+---------------------------------------------------|
| =ORG_CAPTURE=        | url, title, text, raise | Open org-capture                                  |
| =ORG_ROAM_CAPTURE=   | url, title, text, raise | Open org-roam capture                             |
| =EWW=                | url, raise              | Open URL in eww                                   |
| =SAVE_PAGE=          | url, title, html        | Save main content to =browsel-www-archive-dir=      |
| =CHATGPT=            | url, title, turns       | Save conversation to =browsel-chatgpt-dir=          |
| =YOUTUBE=            | url, title, text, raise | Capture into =browsel-youtube-videos-file=          |
| =YOUTUBE_TRANSCRIPT= | url, title, raise       | Save transcript to =browsel-youtube-transcript-dir= |

Capture handlers also prime three dynamic vars consumable from
org-capture templates: =(browsel-get-url)=,
=(browsel-get-title)=, =(browsel-get-selection)=.

** Emacs → browser

| Name               | Args                           | Returns                               |
|--------------------+--------------------------------+---------------------------------------|
| =GET_ALL_TABS=       | (none)                         | array of Chrome Tab plists            |
| =GET_ACTIVE_TAB=     | (none)                         | array (active tabs of current window) |
| =OPEN_TAB=           | =:url=                           | the created Tab                       |
| =FOCUS_TAB=          | =:id=, optional =:focusWindow=     | ={:status "ok"}=                        |
| =CLOSE_TAB=          | =:id=                            | =null=                                  |
| =EVAL_IN_ACTIVE_TAB= | =:code=, optional =:world=, =:tabId= | =:result= = array of =InjectionResult=    |

=InjectionResult= is =(:documentId "..." :frameId 0 :result <value>)=.

* All Emacs-side configuration variables

| Variable                                     | Purpose                                                                      |
|----------------------------------------------+------------------------------------------------------------------------------|
| =browsel-port=                                 | WS server port (default 9130)                                                |
| =browsel-host=                                 | bind address.  =local= = 127.0.0.1; =nil= = all interfaces                       |
| =browsel-request-timeout=                      | seconds before async req aborts (default 10)                                 |
| =browsel-debug=                                | when non-nil, log every WS frame to *browsel*                                  |
| =browsel-org-capture-key=                      | template key passed to =org-capture= for =ORG_CAPTURE=; =nil= = interactive picker |
| =browsel-pandoc-executable=                    | =pandoc= binary used by save-page and chatgpt                                  |
| =browsel-www-archive-dir=                      | directory where save-page writes archives (set this)                         |
| =browsel-chatgpt-dir=                          | directory where save-chatgpt writes conversations (set this)                 |
| =browsel-youtube-api-key=                      | YouTube Data API v3 key, for video metadata                                  |
| =browsel-youtube-videos-file=                  | org file the YouTube capture appends to (set this)                           |
| =browsel-youtube-transcript-dir=               | directory where YouTube transcripts are written (set this)                   |
| =browsel-youtube-transcript-yt-dlp-executable= | =yt-dlp= binary used to fetch subtitles                                        |

The bundled =defvar= defaults assume a particular directory layout;
the path variables should be set explicitly per installation.

* Extending the Emacs side

** Add a new browser → Emacs handler

#+begin_src emacs-lisp
(browsel-register-handler
 "OPEN_IN_FIREFOX"
 (lambda (payload)
   (start-process "ff" nil "open" "-a" "Firefox" (plist-get payload :url))
   '((status . "ok") (message . "opened in Firefox"))))
#+end_src

Pair it with a config menu entry whose =request= matches the name.
For long-running actions, use the respond-fast-then-defer pattern:
return =((status . "ok"))= immediately, schedule the real work with
=(browsel-defer #'do-it payload)=.

** Add a new Emacs → browser handler

If the =chrome.*= API you want is a regular function, add a config
=handlers= entry — no JS needed.  If it needs an adapter (multiple
args, transforming the payload), add a new entry to =SHAPE_ADAPTERS=
in =extension/src/handlers.js= and reference it via =args-shape=.

** Emacs-side primitives

| Call                                                           | Notes                                                                             |
|----------------------------------------------------------------+-----------------------------------------------------------------------------------|
| =(browsel-request NAME PAYLOAD &optional CLIENT)=                | Sync.  Blocks with =accept-process-output=.  Do not call from inside a WS callback. |
| =(browsel-request-async NAME PAYLOAD CALLBACK &optional CLIENT)= | Async.  Returns request id.  Always safe.                                         |
| =(browsel-register-handler NAME FN)=                             | Register a browser → Emacs handler.  Overwrites on duplicate.                     |
| =(browsel-connected-clients)=                                    | List of currently connected client names (newest first).                          |
| =(browsel-defer FN &rest ARGS)=                                  | Schedule FN to run on the next idle tick — for respond-fast-then-defer.           |

* Extending the Chrome extension side

** =payload.js= — built-in payload kinds

Each kind is a =case= in =gatherPayload=.  For irregular shapes prefer
the object form (=tab-message=) which is config-only and doesn't
require an edit here.

** =handlers.js= — shape adapters

Functions in =SHAPE_ADAPTERS=.  Each takes the request payload and
calls a =chrome.*= API.  Document the expected payload at the top of
the adapter.

** =content.js= — generic page-side relay

Web pages can post a message to fire any Emacs handler:

#+begin_src js
window.postMessage(
  { source: "browsel", name: "EWW", payload: { url: "..." } },
  "*"
);
#+end_src

The content script forwards to the service worker, which dispatches.
Useful for page-side helpers.

** =consent.js= — eval gate

The =user-script= shape adapter is gated by =ensureConsent(tabId,
code)=.  The first eval on a tab displays the in-page overlay; the
result is stored in =chrome.storage.session= keyed by tab id, with an
expiry field (=null= = until tab close, ms epoch = expires at that
time).

** Build pipeline

=scripts/build-manifest.py= reads =config.json= and, given
=--target chrome= or =--target firefox=, emits the matching
=build/<target>/manifest.json= — deep-merging =config.extension=
with =config.extensionTargets.<target>=, adding =content_scripts=
from =config.contentScripts= and =commands= collected from
=menus[].command=.

=make lint= verifies every JS file's =import= statements reference
files that exist in the build tree.  Forgetting to add a new file to
=SHARED_FILES= or to the per-target overlay manifests as "Service
worker registration failed.  Status code: 3".

=make help= lists the targets.

* Build / verify cycle

#+begin_src bash
# Extension (default builds every known target)
cd extension && make           # same as `make all`: chrome + firefox
cd extension && make chrome    # only build/chrome/ (Manifest V3)
cd extension && make firefox   # only build/firefox/ (Manifest V2)

# Elisp — byte-compile to catch warnings:
emacs --batch -Q --eval '(require (quote package))' \
  --eval '(package-initialize)' \
  -L . \
  --eval '(dolist (f (list "browsel.el" "browsel-www.el"
                           "browsel-chatgpt.el" "browsel-youtube.el"
                           "browsel-babel.el"))
           (or (byte-compile-file f) (kill-emacs 1)))'

# Reload in a running Emacs:
emacsclient -e '(progn (browsel-stop) (browsel-start))'

# Confirm WS bound and the Chrome extension connected:
lsof -nP -iTCP:9130 -sTCP:LISTEN
emacsclient -e '(format "clients=%d" (length browsel--clients))'
#+end_src

* Debugging surfaces

| Side             | Where                                                         |
|------------------+---------------------------------------------------------------|
| Emacs            | =(setq browsel-debug t)= → =*browsel*= buffer per frame           |
| Emacs            | =*Messages*= for handler success/error                          |
| Chrome extension | =chrome://extensions= → Details → Inspect views: service worker |
| Chrome extension | =chrome://extensions= → Details → Inspect views: offscreen.html |

For non-obvious failure modes (MV3 setIcon paths failing, IPv6
localhost, =defvar= not overwriting, =lexical-binding= breaking
=org-capture-initial=, YouTube PoToken gating, …) see =ai/gotchas.md=.

* Security implementation

Threat model and what's /not/ protected are in =README.org=.  Two
implementation notes belong here:

*Per-tab consent.*  =extension/src/consent.js= stores grants in
=chrome.storage.session= as
=consentByTab[<tabId>] = { expiry: <ms> | null }=.  =null= = until
tab close; ms = wall-clock expiry.  =ensureConsent= is the single
gate, called from the =user-script= shape adapter.
=chrome.tabs.onRemoved= removes entries (tab ids are reused over time).

*Toolbar icon swap.*  =chrome.action.setIcon({path: ...})= silently
fails inside MV3 service workers.  =background.js= and =handlers.js=
build =ImageData= from the PNGs via =OffscreenCanvas= instead.
